# 설계: 인터뷰형 입력 흐름 (백신 챙겨 / Vaxkeep)

- 날짜: 2026-07-07
- 상태: 승인됨 (구현 계획 대기)
- 대상 파일: `index.html` (단일 파일 정적 앱)

## 1. 배경 / 문제

현재 입력은 정적 폼이다. 나이·성별·기저질환·이력·생년월·여행을 한 화면에 동시에 보여줘 **"한꺼번에 물어봐서 부담"**된다(사용자 확인된 핵심 불만). 반대로 사용자는 **빠른 효과**(입력 즉시 결과)와 **나중에 수정 가능**도 원한다.

## 2. 목표 / 비목표

**목표**
- 화면에 **동시에 보이는 질문은 항상 1개**로 부담 최소화.
- **나이 하나만 답해도 즉시 결과** — 결과는 이후 사라지지 않고 갱신만.
- 답변은 언제든 **요약 칩으로 수정** 가능(실시간 반영).
- 기존 규칙 엔진·페이스메이커·여행·유행경보·AI 설명을 **재사용**(로직 변경 없음).

**비목표**
- LLM 대화형 인터뷰(접근 C)는 이번 범위 아님(향후 별도 진입점).
- 규칙/임상 로직 변경 없음. 백신 데이터 변경 없음.
- 백엔드 추가 없음(무서버 정적 유지).

## 3. UX 흐름

1. 홈 → 시작.
2. **질문 1: "나이가 어떻게 되세요?"**(세/개월 토글). 답 → 즉시 결과 표시.
3. 결과 아래로 후속 질문이 **한 장씩** 등장: 성별 → 기저질환 → 접종이력 → (opt-in)여행.
4. 각 답변 → 질문 카드가 상단 **요약 칩**으로 접힘 → **결과 실시간 갱신** → 다음 질문 카드 등장.
5. **"이대로 충분해요"**로 언제든 종료(이미 결과 있음). 요약 칩의 ✎ 로 재수정 → 실시간 반영.
6. 여행은 **"＋ 여행 계획 추가"** 를 눌러야 등장(기본 흐름에서 안 물음).
7. 상단 **"한 문장으로 말하기"(선택)** 로 한 번에 여러 답 채움(휴리스틱, AI_API 설정 시 LLM) → 나머지만 이어서 물음.

**화면 레이아웃**
```
← 백신 챙겨
[63세 ✎] [여성 ✎] [＋조건]        ← 답한 것 = 수정 가능 요약 칩
──────────────────────────
🟢 지금 3가지 챙길 때예요           ← 결과(항상 위, 실시간 갱신)
· 인플루엔자 … / 폐렴구균 …
📅 접종 페이스메이커 …
──────────────────────────
┃ 혹시 만성질환 있으세요?           ← 현재 질문 1개만
┃ [당뇨·심폐][간][면역저하]…
┃ [해당 없음]          [건너뛰기]
정확도 ▓▓▓░░ 더 답할수록 정확        ← 강요 대신 "정확도↑" 피드백
```

## 4. 상호작용 원칙

- 동시에 보이는 질문 1개. 결과는 사라지지 않고 갱신만.
- 모든 답변은 요약 칩으로 수정 가능.
- "정확도" 미터로 "덜 답해도 됨 + 더 답하면 정확"을 강요 없이 전달.
- 나이 미입력 시 결과 없음(질문 1이 게이트).

## 5. 상태 모델 (단일 진실원)

```
state = {
  ageRaw: number|null, unit: 'year'|'month',   // 화면 입력값
  gender: '여'|'남'|'',
  cond: Set<string>,   // 만성질환·간질환·면역저하·임신·없음
  hist: Set<string>,   // 독감·폐렴구균·… (10종)
  birth: 'YYYY-MM'|'',
  travel: 'YYYY-MM-DD'|'', region: ''| 'sea'|...,
  answered: Set<stepId>   // 어떤 질문에 답했는지(요약 칩·진행도용)
}
```
- 파생값 `ageM`(개월)은 `ageRaw`·`unit`에서 계산.
- 프로필 저장/불러오기(localStorage)는 이 `state`를 직렬화.

## 6. 컴포넌트 분해 (기존 `showResult` 리팩터)

| 유닛 | 책임 | 비고 |
|---|---|---|
| `computeGroups(state)` | 규칙 엔진 → {now,later,done} | 현 `showResult` 상단 로직 이관, 로직 불변 |
| `renderResult(groups, state)` | 결과 카드+페이스메이커+여행+유행경보+AI설명 | 현 `showResult` 하단 이관, DOM 대신 state 인자 |
| `STEPS[]` | 질문 순서·각 질문 렌더러/검증 | age→gender→cond→hist→(travel) |
| `renderInterview()` | 요약 칩 + 현재 질문 1개 + 결과(위 두 함수 호출) | 화면 오케스트레이터 |
| `setAnswer(stepId, value)` | state 갱신 → 다음 질문 노출 → renderResult 재호출 | 실시간 갱신 진입점 |
| `editAnswer(stepId)` | 해당 질문을 다시 활성 질문으로 | 요약 칩 ✎ |

- 페이스메이커/여행/유행경보/`explain()`(AI)은 **인자만 state/groups로 받게** 하고 내부 불변.
- 칩 토글 핸들러(`multiToggle` 등)는 DOM aria 대신 `state`를 갱신하도록 조정하거나, DOM을 읽어 state로 동기화하는 얇은 어댑터 사용(구현 계획에서 택1).

## 7. 질문 순서 (STEPS)

1. `age` — 나이(세/개월). **필수·게이트**. 답하면 결과 첫 표시.
2. `gender` — 성별(여/남/건너뛰기). HPV·MMR 여성 분기에 영향.
3. `cond` — 기저질환·상태 칩(+해당 없음/건너뛰기).
4. `hist` — 이미 접종 완료 칩(+건너뛰기).
5. `travel` — opt-in("＋ 여행 계획 추가"). 출발일 + 여행지(권역).

## 8. 엣지 케이스

- 나이만 답하고 종료 → 나이 기반 결과(gender/cond 공백).
- 성별·임신 미응답 → HPV·MMR 여성 분기 생략(첫 결과 대략, 답하면 정교).
- 남성 선택 시 임신 조건 무시(기존 로직 유지).
- 여행 미추가 → 여행 섹션 미표시.
- 홈 복귀 시 state 유지(메모리). 프로필 저장은 명시적.

## 9. 성공 기준

- 첫 화면에 질문 1개(나이)만 보인다.
- 나이 입력 후 1회 상호작용으로 결과가 뜬다.
- 이후 질문에 답할 때마다 결과가 그 자리에서 갱신된다(화면 이동 없음).
- 요약 칩으로 임의 답을 수정하면 결과가 즉시 반영된다.
- 기존 규칙/페이스메이커/여행/AI 설명 결과가 리팩터 전과 동일하다(회귀 없음).
- JS 문법·주요 페르소나(영유아/65세+/임신부/여행) 스모크 통과.

## 10. 범위 밖(향후)

- AI 대화형 인터뷰(접근 C) 별도 진입점.
- 적응형 질문 스킵(예: 65세+면 소아 질문 생략) — 이번엔 고정 순서, 추후 개선.
